Here are 3 simple steps to help get you started...
Step 1: Use the Mediator Configuration tool to enable the webdev node. 
Step 2: Design and create a custom HTML file with some special attributes included in specified HTML tags.
Step 3: Include some Javascript code (described below) in your page.
Step 4: Publish the files to the Mediator.
< previous step | next step >
Step 1

Start the Configuration Tool and find the webdev node under /services/network.
Check Enable and Accept FTP connections.
 
Click image or here to enlarge

Save the configuration
Step 2
Design and create an HTML file using your favorite HTML editor (for example, Dreamweaver).
Insert a node attribute inside the appropriate supported HTML tag elements (see the list below). The node attribute contains the node path of the node value you want to display.

Example:
<span node="/services/time"></span>
<td node="/interfaces/relay1"></td>

Important Note: always use the closing tag "</span> or </td>"

Supported tags:
<span>
<div>
<td>
<img>
<input>
Important Note: The <img> tag needs some additional attributes added to the tag to work correctly.
 
Step 3 

Include the following javascript code inside the <head> tags of your document:

<script language="javascript"src="/webapi/js/sdk2.js"></script>

Include the following javascript code inside the </body> </html> tags of your document:

<script language="javascript">
    var m = new Mediator();
    var page_properties = new PageProperties(m,5000);
    start_refresh_loop(page_properties);
</script>

The script shown above will ask you to login the first time it tries to refresh and then continue to refresh every five seconds.
 
Step 4 

FTP the file of files to the Mediator using webdev as both the username and the default password. 

To learn more about publishing files to the Mediator, visit the Publishing help page.
